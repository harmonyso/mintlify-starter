---
title: "Password Reset Request"
description: "Automated password reset request with identity verification and temporary password delivery."
---

The **Password Reset Request** agent handles password resets for identity provider users. It verifies identity and delivers a temporary password or reset link securely.

## What it does

- Detects password reset requests (typically from tickets)
- Verifies employee identity
- Initiates password reset in the IdP (Okta, Entra, Google Workspace)
- Delivers temporary password or reset link to the user
- Updates the ticket with status

## Triggers

**Event** — Runs when a password reset request is detected (e.g., ticket created with `is_password_reset_request`).

## Approval

Typically does not require approval; identity verification is the primary control. Approval can be configured for sensitive cases.

## Configuration

- **Identity verification** — Method and requirements
- **Identity provider** — Okta, Entra, Google Workspace
- **Delivery channel** — How temporary password or reset link is sent
- **Password policy** — Expiry, complexity (when applicable)

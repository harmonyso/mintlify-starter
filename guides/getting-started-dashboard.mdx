## Welcome to Harmony Dashboard

Welcome to Harmony, your intelligent IT operations platform! Harmony Dashboard is your central command center for managing tickets, assets, software, and automation across your organization. ## What is Harmony Dashboard?

Harmony Dashboard is a comprehensive IT service management (ITSM) platform that brings together ticketing, asset management, automation, and AI-powered workflows into a single, unified interface.

### How Harmony Works

Harmony operates through multiple integrated modules:

1. **Dashboard & Analytics** — Visualize metrics with customizable widgets and pre-built dashboards
2. **Service Desk** — Manage tickets across multiple desks with team-based routing
3. **Asset Management** — Track hardware inventory, software licenses, and compliance status
4. **AI Agents** — Automate routine tasks like password resets, compliance checks, and onboarding
5. **Workflows** — Build custom automation with visual workflow builder
6. **Integrations** — Connect to your existing tools (Slack, Okta, Jamf, Intune, and more)
7. **Knowledge Base** — Centralize documentation and enable AI-powered solution recommendations

## Accessing the Dashboard

### Getting Your Harmony Account

Your Harmony account is set up by your organization's IT administrator. You'll receive:

1. **Welcome Email** — Contains your unique Harmony portal URL (e.g., `yourcompany.harmony.io`)
2. **Login Instructions** — How to authenticate (typically via SSO)
3. **Role Information** — Your assigned permissions (Admin, Agent, or Observer)

<Info>
  Don't have an account yet? Contact your IT administrator or [request a demo](https://harmony.io/demo) if you're evaluating Harmony for your organization.
</Info>

### Finding Your Harmony URL

Your Harmony instance uses a unique subdomain:

```
https://[your-company].harmony.io
```

**Examples:**
- `acme.harmony.io`
- `contoso.harmony.io`
- `startup.harmony.io`

<Tip>
  Bookmark your company's Harmony URL for quick access. You can also add it to your browser's home screen on mobile devices.
</Tip>

### Authentication Methods

Harmony supports multiple authentication methods:

<Tabs>
  <Tab title="SSO (Single Sign-On)">
    **Most Common Method**
    
    Log in using your existing company credentials through:
    - Okta
    - Microsoft Entra ID
    - Google Workspace
    - Any SAML 2.0 or OIDC provider
    
    **How it works:**
    1. Visit your Harmony URL
    2. Click "Continue with SSO"
    3. Authenticate through your company's identity provider
    4. Automatically redirected to Harmony Dashboard
    
    <Info>
      SSO provides the most secure experience and eliminates the need to remember another password.
    </Info>
  </Tab>
  
  <Tab title="Email Magic Link">
    **Passwordless Authentication**
    
    1. Enter your work email address
    2. Check your inbox for a magic link
    3. Click the link to authenticate
    4. Access granted without passwords
    
    <Note>
      Magic links expire after 15 minutes for security.
    </Note>
  </Tab>
  
  <Tab title="Admin Login">
    **For Harmony Administrators**
    
    Admin users can access a special login flow:
    1. Visit `yourcompany.harmony.io/login?admin=true`
    2. Use admin credentials
    3. Access full administrative privileges
  </Tab>
</Tabs>

### Common First-Time Questions

<AccordionGroup>

### Dashboard Navigation Elements

#### Dashboard Dropdown Selector

Click the dashboard name in the header to access:

**Features:**
- **Search** — Filter dashboards by name
- **Dashboard List** — Shows all accessible dashboards
- **Metadata** — Creator and creation date for each dashboard
- **Actions:**
  - ⭐ Star icon — Set as default dashboard
  - 📋 Duplicate — Copy dashboard with all widgets
  - 🗑️ Delete — Remove custom dashboards (system protected)
- **Create New** — Button to create custom dashboard

**System vs Custom Dashboards:**

| Feature | System Dashboard | Custom Dashboard |
|---------|------------------|------------------|
| Editable | ❌ Read-only | ✅ Full control |
| Deletable | ❌ No | ✅ Yes |
| Set as Default | ❌ No | ✅ Yes |
| Duplicatable | ✅ Yes | ✅ Yes |
| Share with Team | ❌ No (universal) | ✅ Yes |

#### Time Range Selector

Filter all dashboard widgets simultaneously:

**Preset Options:**
- **1d** — Last 24 hours
- **7d** — Last 7 days
- **30d** — Last 30 days (default)
- **90d** — Last 90 days
- **Custom** — Pick specific start and end dates

**Behavior:**
- Applies to ALL widgets on the dashboard
- Updates URL query params for bookmarking
- Automatically adjusts data granularity:
  - 1-7 days → Hourly data points
  - 8-30 days → Daily data points
  - 31-90 days → Weekly data points
  - 90+ days → Monthly data points

<Tip>
  Use the time range selector to compare different periods. For example, switch from 7d to 30d to identify longer-term trends.
</Tip>

#### Edit Mode Controls

Transform your dashboard with edit mode:

**Entering Edit Mode:**
1. Click **"Edit"** button in the top header
2. Dashboard enters edit mode
3. Additional controls appear on each widget:
   - **Drag handle** (top-left) — Move widget
   - **Resize handle** (bottom-right) — Adjust size
   - **Remove button** (×) — Delete widget

**In Edit Mode You Can:**
- ➕ Add new widgets
- 🔄 Rearrange widget positions
- 📏 Resize widgets (within constraints)
- ❌ Remove unwanted widgets
- ✏️ Edit dashboard name
- 💾 Save changes
- 🚫 Cancel and revert

**Saving Your Changes:**
1. Click **"Save & Exit"** to persist layout and return to view mode
2. Or click **"Cancel"** to discard all changes

<Warning>
  Changes are NOT automatically saved in Edit Mode. Always click "Save & Exit" before navigating away, or you'll lose your work.

### Using View Mode (Quick Add)

**Quick Add Workflow:**
1. Click **"Add widgets"** in header
2. Widget library opens as a modal
3. Click any widget to add it instantly
4. Widget is automatically positioned and saved
5. Repeat to add more widgets

**Benefits of View Mode Quick Add:**
- ✅ Auto-save (no manual save required)
- ✅ Faster for adding single widgets
- ✅ No accidental layout changes
- ✅ Widget positioned automatically

<Info>
  Quick Add in View Mode only works on custom dashboards. System dashboards don't have Add or Edit buttons.

### Dashboard Best Practices

<AccordionGroup>
  <Accordion icon="minimize" title="Start Simple, Expand Later">
    Begin with 4-6 essential widgets:
    
    **Recommended First Dashboard:**
    - Total Tickets (number)
    - SLA Compliance % (number)
    - Average Response Time (number)
    - Opened vs Resolved Tickets (line chart)
    - Tickets by Status (donut)
    - Tickets by Priority (bar)
    
    As you get comfortable, add more specialized widgets.
  </Accordion>
  
  <Accordion icon="layer-group" title="Organize by Visual Hierarchy">
    Use the grid layout strategically:
    
    **Top Row** → Most important KPIs (number widgets)
    **Middle Rows** → Trend analysis (line/bar charts)
    **Bottom Rows** → Detailed breakdowns (donut charts, tables)
    
    Number widgets automatically fill the top row left-to-right for consistent placement.
  </Accordion>
  
  <Accordion icon="users-between-lines" title="Create Role-Specific Dashboards">
    Don't build one dashboard for everyone:
    
    - **Leadership** → High-level KPIs only (6-8 widgets)
    - **Managers** → Mix of KPIs and trends (10-15 widgets)
    - **Operators** → Detailed breakdowns and real-time status (15-20 widgets)
    
    Use the dashboard dropdown to switch between contexts.
  </Accordion>
  
  <Accordion icon="expand" title="Size Widgets Appropriately">
    Bigger isn't always better:
    
    - **Small (3-4 columns)** → Number widgets, simple metrics
    - **Medium (6 columns)** → Single trend lines, basic breakdowns
    - **Large (9-12 columns)** → Complex multi-series charts, detailed data
    
    Reserve full-width (12 columns) for your most critical visualizations.
  </Accordion>
  
  <Accordion icon="palette" title="Understand Color Coding">
    Widgets use consistent color semantics:
    
    - 🟢 **Green** → Positive, on-track, compliant
    - 🔴 **Red** → Issues, breaches, non-compliant
    - 🟠 **Orange** → Warnings, in-progress, approaching limits
    - 🔵 **Blue** → Neutral data, informational
    - 🟣 **Purple** → AI/automation-related metrics
  </Accordion>
</AccordionGroup>

### Troubleshooting Common Issues

<AccordionGroup>
  <Accordion icon="circle-exclamation" title="I can't log in">
    **Possible causes:**
    - Incorrect URL (verify your company subdomain)
    - SSO configuration issue
    - Network/firewall blocking `*.harmony.io`
    - Expired session (try clearing browser cache)
    
    **Solutions:**
    1. Verify correct URL with IT admin
    2. Try incognito/private browsing
    3. Check network connectivity
    4. Contact Harmony support if issues persist
  </Accordion>
  
  <Accordion icon="circle-exclamation" title="Dashboard shows 'No data available'">
    **Possible causes:**
    - No data exists for selected time range
    - Integrations still syncing (first-time setup)
    - Insufficient permissions
    - API connection issue
    
    **Solutions:**
    1. Try expanding time range to 90 days
    2. Wait 30-60 minutes if integrations just connected
    3. Verify permissions with admin
    4. Check system status page
  </Accordion>
  
  <Accordion icon="circle-exclamation" title="My changes won't save">
    **Possible causes:**
    - Editing system dashboard (read-only)
    - Network interruption during save
    - Browser cache issue
    - Session timeout
    
    **Solutions:**
    1. Verify you're on a custom dashboard (not system)
    2. Check internet connection
    3. Hard refresh (`Cmd+Shift+R` / `Ctrl+Shift+R`)
    4. Log out and back in
  </Accordion>
  
  <Accordion icon="circle-exclamation" title="Widgets are loading slowly">
    **Possible causes:**
    - Large dataset (90+ day range)
    - Many widgets on one dashboard
    - Slow internet connection
    - Server under heavy load
    
    **Solutions:**
    1. Reduce time range to 30 days or less
    2. Remove unused widgets
    3. Check internet speed
    4. Try during off-peak hours
    5. Contact support if consistently slow
  </Accordion>
  
  <Accordion icon="circle-exclamation" title="I don't see menu items I expect">
    **Possible causes:**
    - Insufficient role permissions
    - Feature not enabled for your tenant
    - Feature behind feature flag
    
    **Solutions:**
    1. Verify your role with admin
    2. Request permission upgrade if needed
    3. Contact support to enable feature
  </Accordion>
</AccordionGroup>

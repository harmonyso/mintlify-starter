## Security Controls

### Information Security Management

<AccordionGroup>
  <Accordion title="Security Leadership">
    The Harmony leadership team and Data Privacy Officer (DPO) are responsible for determining and updating information security measures at Harmony.
  </Accordion>

  <Accordion title="Information Security Policies and Procedures">
    Harmony maintains established information security policies and procedures for all employees and contractors. Policies and procedures are approved by management, reviewed at least annually, and made available to all employees via our intranet.
  </Accordion>

  <Accordion title="Background Checks">
    Harmony performs background checks on all new employees in accordance with local laws.
  </Accordion>

  <Accordion title="Employee Confidentiality">
    All employee contracts include a confidentiality agreement.
  </Accordion>

  <Accordion title="Mandatory Security Awareness Training">
    All employees undergo mandatory security awareness training on an annual basis. Certain higher risk roles go through additional training specific for their role and its associated risks, annually.
  </Accordion>
</AccordionGroup>

### Network & Infrastructure Security

<AccordionGroup>
  <Accordion title="Auto Scaling">
    Our infrastructure auto-scales to maintain high availability and support demand.
  </Accordion>

  <Accordion title="Backups and Monitoring">
    On an application level, we produce audit logs for all activity, ship logs to a centralised logging system for analysis. All actions taken on production consoles or in the Harmony application are logged.
  </Accordion>

  <Accordion title="Disaster Recovery">
    Harmony was built with disaster recovery in mind. All of our infrastructure and data are spread across different availability zones and will continue to work should any one of those data centers fail.
  </Accordion>

  <Accordion title="Least Privilege">
    AWS Security Groups employed for our infrastructure are baselined regularly to maintain least privilege. Access is granted to team members for our AWS production environment are baselined on a regular basis to maintain least privilege.
  </Accordion>
</AccordionGroup>

### Data Privacy & Protection

<AccordionGroup>
  <Accordion title="Encryption at Rest">
    We encrypt data at rest using an industry-standard AES-256 encryption algorithm.
  </Accordion>

  <Accordion title="Encryption in Transit">
    Harmony is served 100% over HTTPS. All data sent to or from Harmony is encrypted in transit using 256 bit encryption. Our API and application endpoints are TLS/SSL only.
  </Accordion>

  <Accordion title="Data Retention and Disposal Policies">
    Workspace data is deleted within 30 days of a workspace being electively deleted by its teammates.
  </Accordion>
</AccordionGroup>

### Incident Detection & Response

<AccordionGroup>
  <Accordion title="Code Review">
    Each pull request to the Harmony code repositories must undergo a peer review before it can be accepted and merged.
  </Accordion>

  <Accordion title="Incident Response Process">
    Harmony implements a protocol for handling security events which includes escalation procedures, rapid mitigation and post mortem. All employees are informed of our policies.
  </Accordion>

  <Accordion title="On-call Coverage">
    A member of engineering is on-call 24/7 to respond to alerts and pages. They can escalate directly to a security team member as needed.
  </Accordion>

  <Accordion title="Security Policies">
    Harmony has developed a comprehensive set of security policies covering a range of topics. These policies are updated frequently and shared with all employees.
  </Accordion>

  <Accordion title="Vendor Risk Assessments">
    We formally assess the security posture of all third-party vendors, with a higher bar for those which handle sensitive data or have access to critical systems.
  </Accordion>
</AccordionGroup>

### Product Security

<AccordionGroup>
  <Accordion title="Multi-tenancy Data Protections">
    Safeguards are in place such that data from one Harmony workspace can never be used or displayed within another workspace.
  </Accordion>

  <Accordion title="Password Complexity">
    Harmony enforces a password complexity standard.
  </Accordion>

  <Accordion title="Penetration Testing">
    We engage third-party security experts to perform detailed penetration tests on the Harmony application and infrastructure annually.
  </Accordion>

  <Accordion title="SSO">
    You can configure Harmony with SAML Single Sign-on (SSO) using Okta, Azure AD or another identity SAML provider.
  </Accordion>

  <Accordion title="Upload Scanning">
    High-risk executable files are automatically restricted for all workspaces. Customers can choose which filetypes can be uploaded by users to their workspace. Allowed files are scanned for malware.
  </Accordion>
</AccordionGroup>

### Endpoint Security

<AccordionGroup>
  <Accordion title="Endpoint Encryption">
    All corporate devices are encrypted to protect data in case of loss or theft. They can be remotely wiped to prevent data leakage if a device is compromised or lost.
  </Accordion>

  <Accordion title="Endpoint Management">
    We push updates to employee laptops such that they are on the latest, patched version of their required operating system. We require the use of a managed browser with only an approved set of browser extensions.
  </Accordion>
</AccordionGroup>

## FAQ

<AccordionGroup>
  <Accordion title="How do I report an information security issue or concern?">
    You can report security issues through our dedicated security email at privacy@harmony.io or through our secure portal.
  </Accordion>

  <Accordion title="How is customer data protected?">
    We implement multiple layers of security including encryption at rest and in transit, access controls, and regular security audits.
  </Accordion>

  <Accordion title="Where is customer data stored?">
    Customer data is stored in secure, compliant data centers with our primary cloud provider Amazon Web Services in the USA.
